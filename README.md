# NSEdge-Validation

NSEdge-Validation is a comprehensive simulation and validation testing suite for distributed edge computing environments. It compares the custom physical discrete-event edge simulator, **NSEdge** (built upon NS-3 Direct Code Execution and physical Linux packet stacks), against prominent third-party edge frameworks: **Simu5G**, **EdgeCloudSim**, and **EdgeSimPy**.

## 🌟 The "No Analytical Proxies" Philosophy

A major design choice of this repository was to **abandon analytical math proxies** for baseline comparisons. Many research papers utilize frameworks like EdgeSimPy and arbitrarily append `delay = distance / speed_of_light` math formulas. 

Instead, this codebase features **True Native Integrations**. We injected our payloads directly into the lowest-level abstract layers of each engine:
*   **EdgeSimPy**: Natively queues computing tasks utilizing internal Python `Tick` capacities on edge server objects.
*   **EdgeCloudSim**: Natively routes custom payload objects over Java `CloudSim` topologies measuring true MIPS compute overhead.
*   **Simu5G**: Bypasses simplistic delays and models exact 180KB image offloads using the `TcpBasicClientApp` through an OMNeT++ 5G Standalone NR-Uu millimeter-wave air interface.

All results generated by `dipdce_real_injection.py` extract empirical measurements directly from the resulting JSON traces output by these discrete-event engines.

## 🚀 How to Run Natively (Linux)

### 1. Stage the Dependencies
Run the installation script to download `micromamba`, `OMNeT++ 5.6.2`, `OMNeT++ 6.0.3`, and `INET`:
```bash
./install_dep.sh
```

### 2. Compile the Third-Party Frameworks
The bash scripts inside `/third_party/` automatically activate the virtual environments and execute the `make` builds:
```bash
cd third_party
./build_omnet.sh       # Compiles OMNeT++ 5.6.2
./build_omnetpp6.sh    # Compiles OMNeT++ 6.0.3
./build_inet_603.sh    # Compiles INET 4.5.0
./build_simu5g.sh      # Compiles the Simu5G NR modules
```

### 3. Run the Native Executions
You can trigger the 10-second injection traces across all discrete-event engines via the top-level bash script:
```bash
cd third_party
./run_final_baselines.sh
```

### 4. Generate the Visualization
Finally, run the aggregator wrapper which scrapes the outputted JSON files from the runtimes and plots the comparison chart:
```bash
python3 src/simulators/dipdce_real_injection.py
```

## 🐳 How to Run via Docker

To avoid polluting your host machine, you can run the entire evaluation natively within an isolated container.

### 1. Build the Docker Image
Because the `Dockerfile` automatically downloads and compiles the heavy OMNeT++, INET, and Simu5G environments, this build process can take 10-20 minutes depending on your CPU:
```bash
docker build -t nsedge-validation .
```

### 2. Run the Container
Once the container is built, the environment is fully staged and compiled. You can immediately drop into the container and execute the final simulation pipeline:
```bash
docker run -it nsedge-validation

# Inside the container, directly run the native extraction tool:
python3 src/simulators/dipdce_real_injection.py
```
