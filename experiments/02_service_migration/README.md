# Service Migration Experiment (Follow Me at the Edge)

This directory isolates all code, data, and plots for the service migration validation.
The experiment models the "Follow Me at the Edge" (Ouyang et al.) policy, involving a 1 GB state transfer via pre-copy live migration. 

It compares the performance of the proposed **NSEdge** simulator against a **physical emulation testbed** (using Linux `tc netem`).

## Contents
- `results/`: Contains the metrics (`ns3_tasks.csv`, `ns3_tasks_5g.csv`, `physical_tasks.csv`) generated by the simulator and physical testbed.
- `plot_migration.py`: Python script used to parse the CSV files and generate the comparative scatter plot.
- `run_migration_physical.sh`: Shell script to execute the physical emulation testbed.

---

## 1. Running the NS-3 Simulation (NSEdge)

The NS-3 simulation models the migration policy using realistically sized buffers and TCP windowing. 
To ensure the simulation aligns with realistic Linux TCP behaviors, the `ns3-migration-scenario.cc` script explicitly increases `TcpSocket::SndBufSize` and `RcvBufSize` to 4 MB, sets `SegmentSize` to 1440, and uses an `InitialCwnd` of 10.

From the root of `NSEdge-Validation`, run the following steps to execute the simulations and save the results:

**Execution (Ethernet Baseline):**
```bash
cd ../ns-3
./ns3 run 'ns3-migration-scenario --tech=ethernet'
cp results-migration/tasks.csv ../NSEdge-Validation/experiments/02_service_migration/results/ns3_tasks.csv
```

**Execution (5G-NR):**
You can also run the migration scenario utilizing a 5G-NR sub-6 GHz link to demonstrate the impact of 5G transport characteristics on the migration performance.
```bash
./ns3 run 'ns3-migration-scenario --tech=5g'
cp results-migration/tasks.csv ../NSEdge-Validation/experiments/02_service_migration/results/ns3_tasks_5g.csv
cd ../NSEdge-Validation
```

---

## 2. Running the Physical Emulation Testbed

The physical testbed runs two edge worker processes and a state receiver locally, shaping the `lo` loopback interface using `tc qdisc` and `htb` to mirror the delay/bandwidth conditions of the NS-3 simulation.

**Execution:**
From the root of `NSEdge-Validation`:
```bash
cd experiments/02_service_migration
./run_migration_physical.sh
cd ../..
```
*(The `run_migration_physical.sh` script automatically launches the Python processes, handles the `tc` shaping, runs the orchestrator, and saves the output directly to `experiments/02_service_migration/results/physical_tasks.csv`)*

---

## 3. Generating the Plots

Once all three CSVs (`ns3_tasks.csv`, `ns3_tasks_5g.csv`, `physical_tasks.csv`) are generated in the `results/` folder, you can generate the plots using the provided Python script.

From the root of `NSEdge-Validation`:
```bash
cd experiments/02_service_migration
/proj/oasees-PG0/net4hpc/.venv/bin/python3 plot_migration.py
cp results/migration_response_time*.png ../../figs/
cd ../..
```

This will produce `migration_response_time.png` (Ethernet) and `migration_response_time_5g.png` (5G) and copy them to the global `figs/` directory.
